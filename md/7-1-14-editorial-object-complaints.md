#### <a id="redaktion.reklamation"></a>7.1.14. Editorial Object Complaints

Incorrect translations of editorial objects can be contested. It should be noted, that the editorial object to be disputed belongs to the individual team and a valid order context is set. The editorial object must also be risk released for this and be translated into the target language being disputed. If these conditions are met, set the desired target language (1) and select "Make a translation complaint" (2) in the read-only editor context menu (see [Figure 198, “Make a Translation Complaint”](7-1-14-editorial-object-complaints.md#figure.redaktion.reklamation "Figure 198. Make a Translation Complaint")).

It is also possible to make a complaint about a translation in an older version than the current version. To do this, the desired version must then be selected in the history.

<a id="figure.redaktion.reklamation"></a>

![Image: make a translation complaint](images/uebersetzung_reklamieren.jpg)

**Figure 198. Make a Translation Complaint**

Then the dialog to be viewed will open in [Figure 199, “Complaint Dialog”](7-1-14-editorial-object-complaints.md#figure.redaktion.reklamationsdialog "Figure 199. Complaint Dialog"). There you must give a reason for the complaint and its source and you can attach files, for example screenshots. These attachments are transferred with the complaint package to the translator. Only files whose names consist of ASCI characters are accepted as attachments. If files with other characters in the file name are selected, they will not be included in the list of attachments and a message will inform the user of this. There are various options for the source of the complaint, depending on whether it concerns a test sequence or a supplementary document. For supplementary documents, the document content, the display name, or both can be specified. For test sequences, you can select between the test sequence content or the tester name/description or display name.

<a id="figure.redaktion.reklamationsdialog"></a>

![Image: complaint dialog](images/reklamationsdialog.jpg)

**Figure 199. Complaint Dialog**

The complaint will begin when the dialog is confirmed (see [Section 6.4.10, “Translation Disputes”](6-4-10-translation-disputes.md "6.4.10. Translation Disputes")).

A special feature is the complaint of measured values in measured value tables (see [Section 7.6.2, “Creating and Changing Measured Value Tables”](7-6-2-creating-and-changing-measured-value-tables.md "7.6.2. Creating and Changing Measured Value Tables")). Here you can start the complaint of a measured value directly at the measured value. The selected measured value OCMAS text is then transferred directly to the reason for complaint field in the complaint dialog. All of the OCMAS IDs used in the table are listed in the complaint package.
