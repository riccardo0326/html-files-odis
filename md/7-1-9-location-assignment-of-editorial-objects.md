#### <a id="redaktion.standort"></a>7.1.9. Location Assignment of Editorial Objects

All editorial objects are archived in a central database for all locations. It is possible to see and use objects from other locations. There is the location code to clearly differentiate editorial objects with the same name.

This location code is visible in the following displays:

- Navigation/library trees in the editing view

  - Equipment network navigator
  - Knowledge base navigator
  - Function library navigator
  - Control module tree navigator
  - DTC memory navigator
  - Vehicle project navigator
  - Supplemental document navigator
  - XML template navigator
- Selection dialog for editorial objects
- List of editorial objects in order steps, hotfixes to order management

For the displays listed here, there is a separate button in the settings (see [Section 9.1, “Settings”](9-1-settings.md "9.1. Settings")) to select if the location code is visible or not. The location codes for editorial objects are displayed as a default setting. You can adjust this setting yourself. The applied settings are stored locally on the computer, and are set correspondingly again the next time the ODIS Creator application starts.

The location code is also displayed in the object editor (see [Figure 187, “Location Code in Object Editor”](7-1-9-location-assignment-of-editorial-objects.md#figure.redaktion.standortkuerzel "Figure 187. Location Code in Object Editor")). It cannot be turned off there. Location codes cannot be edited. The field serves exclusively as the display.
