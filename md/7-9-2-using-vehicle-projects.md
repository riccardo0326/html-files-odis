#### <a id="fahrzeugprojekte.verwendung"></a>7.9.2. Using Vehicle Projects

With the import of the control modules (see [Section 7.7, “Control Module Tree”](7-7-control-module-tree.md "7.7. Control Module Tree")), these are already provided with information about the respective vehicle projects and are thus already assigned to the individual vehicle projects in the vehicle project tree view. Therefore, the vehicle projects contain all of the control modules used there in the **used** relationship ([2](7-9-vehicle-project-tree.md#figure.fahrzeugprojekt "Figure 291. Vehicle Project Tree") in [Figure 291, “Vehicle Project Tree”](7-9-vehicle-project-tree.md#figure.fahrzeugprojekt "Figure 291. Vehicle Project Tree")). Control modules may be sorted this way in vehicle projects.

The use of an **un**released control module description or an **un**released vehicle project means that the object being used will not be confirmed as completed, but rather only a consistency check test result is generated. This also applies to the indirect use, if a released control module description should have a use relationship to an unreleased vehicle project. Otherwise, the defined consistency rules apply to the respective editorial object type.
