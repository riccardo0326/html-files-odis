#### <a id="beschreibung.rollen"></a>3.3.1. Description of Roles

The separate roles are defined as follows.

<a id="table.rollen.teams"></a>

<table border="1" summary="Roles and Teams"><colgroup><col align="left"/><col align="left"/></colgroup><thead><tr bgcolor="#EEEEEE"><th align="left"><span class="bold"><strong>Role / Team</strong></span></th><th align="left"><span class="bold"><strong>Description</strong></span></th></tr></thead><tbody valign="top"><tr><td align="left" valign="top"><span class="bold"><strong>Editing coordinator</strong></span></td><td align="left" valign="top">An editing coordinator is responsible for order management. He or she creates new orders (see <a class="xref" href="6-2-orders.md" title="6.2. Orders">Section 6.2, “Orders”</a>) and order steps and decides to accept or reject them. The editing coordinator also commissions and distributes the task packages for processing the editorial objects to be created and is responsible for monitoring the completion deadlines. The editing coordinator is not responsible for the quality of the outcome.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Editing coordinator team</strong></span></td><td align="left" valign="top">The editing coordinators in an editing location are included in an editing coordinator team.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Team</strong></span></td><td align="left" valign="top">The addressee of an order step is always a team. The team manages the responsibility for a commissioned order step and the sole responsibility for the editorial objects to be edited. Currently the following teams are designated to edit order steps in ODIS Creator: authoring team, translation team.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Author</strong></span></td><td align="left" valign="top">An author is responsible for processing editorial objects, that are assigned to them within the team. The author is responsible for the quality and the timely completion of the issue.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Authoring team</strong></span></td><td align="left" valign="top">An authoring team edits the editorial objects in the task area it is responsible for. Which author in the team works on a (commissioned) editorial object is decided independently by the members of the team.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Read-only author</strong></span></td><td align="left" valign="top">A read-only author only has read-only access to the ODIS Creator system. The areas that are also available for authors can be viewed by read-only authors. The read-only authors can see objects that have the status "Confirmed completed" or "Released" in the editors, however they cannot change them. The function test editor is also completely usable (including ODIS debug). The read-only author can preview supplemental documents and XML templates as well as the display of values from a measured values table.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Read-only author team</strong></span></td><td align="left" valign="top">A read-only author team includes one or more read-only authors and only has read-only access to the ODIS Creator system.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Translation coordinator</strong></span></td><td align="left" valign="top">A translation coordinator is responsible for the translation management (see <a class="xref" href="6-4-translation.md" title="6.4. Translation">Section 6.4, “Translation”</a>). Associated with this is the commissioning, scheduling and scope statement for the translation of editorial objects in ODIS Creator.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Translation coordinator team</strong></span></td><td align="left" valign="top">The translation coordinators are combined in a translation coordinator team.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>EU5 export coordinator</strong></span></td><td align="left" valign="top">An EU5 export coordinator is responsible for the EU5 export (see <a class="xref" href="8-9-eu5-export.md" title="8.9. EU5 Export">Section 8.9, “EU5 Export”</a>). Associated with this is the definition, validation and download of EU5 export logs in ODIS Creator.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>EU5 export coordinator team</strong></span></td><td align="left" valign="top">The EU5 export coordinators are combined in an EU5 export coordinator team.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Text administrator</strong></span></td><td align="left" valign="top">
<p>The text administrator manages default texts in ODIS Creator (see <a class="xref" href="5-5-default-texts.md" title="5.5. Default Texts">Section 5.5, “Default Texts”</a>). The text administrator creates and edits text groups and default texts; they can deactivate or activate them.</p>
<p>The text administrator <span class="bold"><strong>only</strong></span> available for the Wolfsburg location.</p>
</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Text administrator team</strong></span></td><td align="left" valign="top">The ODIS Creator text administrators are combined in a text administrator team.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Location administrator</strong></span></td><td align="left" valign="top">A location administrator performs location-specific administration tasks for example, the creation of users (see <a class="xref" href="5-2-6-location-data.md" title="5.2.6. Location Data">Section 5.2.6, “Location Data”</a>).</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Administrator team location</strong></span></td><td align="left" valign="top">The ODIS Creator location administrators are combined in a location administrator team.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Group administrator</strong></span></td><td align="left" valign="top">A group administrator performs group-specific administration tasks (see <a class="xref" href="5-2-1-group-data.md" title="5.2.1. Group Data">Section 5.2.1, “Group Data”</a>).</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Group administrator team</strong></span></td><td align="left" valign="top">The ODIS Creator group administrators are combined in a group administrator team.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Cooperation export coordinator</strong></span></td><td align="left" valign="top">A cooperation export coordinator is responsible for the cooperation export. This includes the definition, validation, and download of cooperation export logs in ODIS Creator.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Cooperation export coordinator team</strong></span></td><td align="left" valign="top">The cooperation export coordinators are combined in a cooperation export coordinator team.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Inspector</strong></span></td><td align="left" valign="top">A tester is responsible for Java box testing. This includes testing (accepting or declining) and completing the testing of Java boxes used in ODIS Creator.</td></tr><tr><td align="left" valign="top"><span class="bold"><strong>Testing team</strong></span></td><td align="left" valign="top">The testers are combined in a testing team.</td></tr></tbody></table>
