### <a id="baselines.rollout"></a>8.5. Rollout Baseline

Editorial objects will only be a component in a production baseline if their market introduction date has been reached. So that you can check if the contents match before this deadline, you have the option to create a rollout baseline. This rollout baseline contains editorial objects, that are currently released and would be a component of a production baseline at the specified deadline. Unlike production baselines, rollout baselines are always full.

For rollout baselines, the target folder for the results is specified in the central configuration by the group administrator. The rollout baselines have the same structure as production baselines - the zip archive with the provided object descriptions and contents. The defined target folder applies as the basis for this. To structure the filing, corresponding sub-folders are created when necessary.

If available, the translations in the rollout baseline are exported for all exported editorial objects. In an additional language package, the translation-relevant texts of the respective editorial object is exported in the source language for each editorial object contained in the preparation, in which the applicable editorial object was created. With this, texts that are not translated are displayed in their original language in the processing system.

The version number of a rollout baseline is independent from the production baselines. However, it is built and increased in the same way.
