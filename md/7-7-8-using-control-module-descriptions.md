#### <a id="sgbaum.verwendung"></a>7.7.8. Using Control Module Descriptions

Other editorial object types can use data from unreleased control module descriptions (see [Section 7.7.1, “Importing Control Module Descriptions”](7-7-1-importing-control-module-descriptions.md "7.7.1. Importing Control Module Descriptions")) and vehicle projects ([Section 7.9.1, “Importing Vehicle Projects”](7-9-1-importing-vehicle-projects.md "7.9.1. Importing Vehicle Projects")) (such as [diagnostic objects](7-5-2-diagnostic-objects.md "7.5.2. Diagnostic objects"), [equipment network nodes](7-4-equipment-network.md "7.4. Equipment Network")). The use of one of these objects indicates that the editorial object being used may not be **confirmed as completed**.

The use of an **un**released control module description or an **un**released vehicle project means that the object being used will not be confirmed as completed, but rather only a consistency check test result is generated. This also applies to the indirect use, if a released control module description should have a use relationship to an unreleased vehicle project. Otherwise, the defined consistency rules apply to the respective editorial object type.
