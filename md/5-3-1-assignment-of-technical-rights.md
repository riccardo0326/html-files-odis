#### <a id="funktionale.rechte"></a>5.3.1. Assignment of Technical Rights

##### <a id="zuordnung.perspektiven"></a>5.3.1.1. Assignment of Views

The most basic functional right for a user role is which [views](3-2-1-views-bar.md "3.2.1. Views Bar") they are allowed to use. This assignment is performed in the **Rights management navigator** ([Figure 60, “Rights Management Navigator”](5-3-1-assignment-of-technical-rights.md#administration.rechte "Figure 60. Rights Management Navigator")) in a tree structure, which contains all of the available roles on the first level. The permitted views for the respective roles and an entry that is always available (**General function rights**) are displayed on the second level.

<a id="administration.rechte"></a>

![Image: rights management navigator](images/admin_rechte.png)

**Figure 60. Rights Management Navigator**

The **Views...** menu item can be accessed via the [context menu](2-2-general-definitions-and-glossaries.md#kontextmenu) for a role displayed in the **rights management navigator** ([Figure 61, “Views Context Menu Item”](5-3-1-assignment-of-technical-rights.md#figure.perspektiven.oeffnen "Figure 61. Views Context Menu Item")). By activating or deactivating [checkboxes](2-2-general-definitions-and-glossaries.md#kontrollfeld), additional views can be assigned to roles, or views previously allowed can be revoked ([Figure 62, “Available Views”](5-3-1-assignment-of-technical-rights.md#figure.perspektiven "Figure 62. Available Views")).

<a id="figure.perspektiven.oeffnen"></a>

**Figure 61. Views Context Menu Item**

##### <a id="zuordnung.funktionale.rechte"></a>5.3.1.2. Allocation of Functional Rights (Function Editor)

In order to display the allocation of rights to the individual functions for a roll, the corresponding entry in the rights management [navigator](3-2-3-navigator.md "3.2.3. Navigator") must be clicked on ([Figure 60, “Rights Management Navigator”](5-3-1-assignment-of-technical-rights.md#administration.rechte "Figure 60. Rights Management Navigator")). The corresponding **function editor** will open for display in the right section ([Figure 63, “Function Editor”](5-3-1-assignment-of-technical-rights.md#figure.funktion "Figure 63. Function Editor")). To change the allocation of rights, the editor must be opened in [edit mode](3-2-5-editor.md "3.2.5. Editor").

Another editor can be displayed using the rights management navigator or additional editors can be opened for editing. The role and view, that the function editor is opened in, is displayed at the top of the function editor. This display cannot be changed.

<a id="figure.funktion"></a>

![Image: function editor](images/funktion.png)

**Figure 63. Function Editor**

By activating or deactivating [checkboxes](2-2-general-definitions-and-glossaries.md#kontrollfeld), additional functional rights can be given to or removed from a role ([Figure 63, “Function Editor”](5-3-1-assignment-of-technical-rights.md#figure.funktion "Figure 63. Function Editor")). The function names cannot be changed and only internal names are available in ODIS Creator. This is generally an abbreviation of the English name of the full function name.

In some tables there are **References to object types** in the right column, that are allocated to the respective function. These references are only active if the [Editor being edited](3-2-5-editor.md "3.2.5. Editor") is open.
