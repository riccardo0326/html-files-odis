#### <a id="efa.externe.verweise"></a>7.17.2. External References

External references can be stored for all editorial objects and can be defined in the form of EFA codes on diagnostic objects, supplemental documents and hotspots (see [Figure 593, “External References”](7-17-2-external-references.md#figure.efa.verwendet "Figure 593. External References")). The "EFA" type is to be used for the editorial recording, editing and display of EFA codings as external references. This can be selected in the context menu for the "used" table above the listed editorial objects.

<a id="figure.efa.verwendet"></a>

![Image: external references](images/verwendet.png)

**Figure 593. External References**

After selecting the „EFA“ type, the „EFA Editor“ will open (see [Figure 594, “EFA Editor”](7-17-2-external-references.md#figure.efa.editor "Figure 594. EFA Editor")). This can be portrayed in two different facets. In supplemental documents and hotspots, the external reference can only be recorded or edited in the EFA coding format. When using the EFA editor at a diagnostic object, this also contains control elements to select a variant rule in addition to the control elements to select an EFA code.

<a id="figure.efa.editor"></a>

![Image: EFA editor](images/efa-editor.png)

**Figure 594. EFA Editor**

In both dialog variants, the EFA coding assist can be accessed using the „EFA coding assist“ button. The selection result is transferred from the EFA coding assist to the inactive „EFA code“ text field.
