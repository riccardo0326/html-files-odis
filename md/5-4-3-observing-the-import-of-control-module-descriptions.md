#### <a id="sgbimport.view"></a>5.4.3. Observing the Import of Control Module Descriptions

Your newly created control module description import is displayed in the control module import navigator. If the import has not started yet, it will have the status "Created" (![](images/sgb_created.png)). Since only one import of control module descriptions may run in the system, the import you created will be performed by the system at a later time. After starting, the import status is "Running". During phase 1 (see [Section 5.4.2.1, “Phase 1: Checksum Check”](5-4-2-control-module-import-phases.md#sgbimport.start.phasen.pruefsumme "5.4.2.1. Phase 1: Checksum Check")), the import status is “In progress (check)”. In phase 2 (see [Section 5.4.2.2, “Phase 2: File Import”](5-4-2-control-module-import-phases.md#sgbimport.start.phasen.importderdateien "5.4.2.2. Phase 2: File Import")), the import status is “In progress (file import)”. This will be indicated in the control module import navigator with the ![](images/sgb_in_progress.png) . If an error has already been detected during processing (which allows the processing of the rest of the data to continue), the status changes to “Running with errors (import of the files)”. This will be indicated in the control module import navigator with the ![](images/sgb_in_progress_partial.png) . During the import, the current status will also be displayed in the job monitor (see [Section 5.2.6, “Location Data”](5-2-6-location-data.md "5.2.6. Location Data")).

During the third phase, the status of the import of the preceding phase is adopted and switched accordingly to “Running (relationship processing)” or “Running with errors (relationship processing)”. Since the import continues to run, the clock is also shown as the main symbol in this phase ![](images/sgb_in_progress_partial.png) for “Running with errors (relationship processing)” or ![](images/sgb_finished_in_progress.png) for “Running (relationship processing).” Once the import is complete, it has the final status “Completed” if no errors have occurred (![](images/sgb_finished.png)) or “Completed with errors” (![](images/sgb_partial_finished.png)).

If an error occurs when importing the control module descriptions that does not permit any further processing, the import receives the status “Error” (![](images/sgb_failed.png)).

In the **Control module import properties** view (see [Figure 65, “Control module import view”](5-4-importing-control-module-descriptions.md#figure.sgbimport.navigator "Figure 65. Control module import view")), you will see the creator, the time of creation, and the import start and end time next to the status.

While control module descriptions are importing, the old, current condition of the control module descriptions is displayed in the editor. The new status is displayed only after the import is complete (see [Section 7.7.1, “Importing Control Module Descriptions”](7-7-1-importing-control-module-descriptions.md "7.7.1. Importing Control Module Descriptions")).
