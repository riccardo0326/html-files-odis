#### <a id="redaktionsobjekt.ueberarbeiten"></a>7.1.2. Revising an Editorial Object

An editorial object can be revised, if it is in either the "In processing" status (see [Section 7.1.3.1, “"In Processing" Status”](7-1-3-editorial-object-status.md#redaktion.status.in_bearbeitung "7.1.3.1. \"In Processing\" Status")), the "Confirmed completed" status (see [Section 7.1.3.2, “"Confirmed Completed" Status”](7-1-3-editorial-object-status.md#redaktion.status.fertiggemeldet "7.1.3.2. \"Confirmed Completed\" Status")), or is released. In the first two conditions, you can revise it yourself if you are a member of the team that performs revisions on the editorial object.

For a released editorial object, you must belong to a brand team, which is also associated with the team responsible for the editorial object. Only the teams from this brand are allowed to revise a released editorial object. In the scope of the revision, a new version of the editorial object is also always created. This occurs automatically through the system, if you open a released editorial object to revise it. This will be triggered either by double clicking on the editorial object or by selecting **File >> Edit**.

After the released editorial object has been opened for editing, you can still delete the new version just created, if you have not saved the editorial object yourself yet. This process is automatically assumed by the system, if you cancel the revision of the editorial object and close the editor without saving.
