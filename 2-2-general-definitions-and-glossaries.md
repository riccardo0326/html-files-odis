### <a id="definitionen.begriffe"></a>2.2. General Definitions and Glossaries

Fundamental and recurring terms and operations are explained in more detail in this section.

<a id="arretiertaste"></a>**Lock button (toggle button)**
:   Lock buttons are used by clicking on them, such as [Figure 5, “Lock Button”](2-2-general-definitions-and-glossaries.md#figure.arretiertaste "Figure 5. Lock Button"). They will be activated or deactivated by clicking on them and will then maintain the selected status until the next input.

    <a id="figure.arretiertaste"></a>

    ![Image: lock button](images/kontrollfeld.png)

    **Figure 5. Lock Button**

<a id="auswahlfeld"></a>**Selection field (combo box) and selection list**
:   The program makes entries which the user can select available through so-called selection fields ([Figure 6, “Selection Field”](2-2-general-definitions-and-glossaries.md#figure.auswahlfeld "Figure 6. Selection Field")).

    <a id="figure.auswahlfeld"></a>

    ![Image: selection field](images/Verknuepfungsziel.png)

    **Figure 6. Selection Field**

    If a selection field ([Figure 6, “Selection Field”](2-2-general-definitions-and-glossaries.md#figure.auswahlfeld "Figure 6. Selection Field")) is clicked on, a selection list will open ([Figure 7, “Selection List”](2-2-general-definitions-and-glossaries.md#figure.bildlauf.auswahlfeld "Figure 7. Selection List")). Through the arrow keys ![](<images/Pfeil hoch.png>) or ![](<images/Pfeil runter.png>) can be navigated to the top or to the bottom. Alternatively, the scroll bar (marked in green) can be moved. Click on the desired entry to apply it.

    <a id="figure.bildlauf.auswahlfeld"></a>

    ![Image: scrolling selection field](images/Standortauswahl.png)

    **Figure 7. Selection List**

<a id="auswahlmenu"></a>**Selection Menu**
:   A selection menu is a list of multiple [buttons](2-2-general-definitions-and-glossaries.md#schaltflaeche), that is opened by clicking a superordinate button (for example [Figure 8, “Selection Menu”](2-2-general-definitions-and-glossaries.md#figure.auswahlmenu "Figure 8. Selection Menu")). The buttons in the selection menu are accessed by clicking on them. If you click in an area outside of the open selection menu, the selection menu will close.

    <a id="figure.auswahlmenu"></a>

    ![Image: selection menu](images/auswahlmenu.png)

    **Figure 8. Selection Menu**

<a id="dialogmaske"></a>**Dialog screen**
:   An example for a dialog screen is the date input ([Figure 9, “Dialog Screen Date”](2-2-general-definitions-and-glossaries.md#figure.auswahlfeld.datum "Figure 9. Dialog Screen Date")). Here the month and year can be changed by clicking on the month/year. However, the day is applied by clicking directly on it. By pressing the OK button, the selected value will be applied and the dialog will close. By pressing Cancel, the dialog will close without applying the value. In a standard scenario, the dialogs act as so-called modal dialogs. This means the application is blocked for entries in other elements as long as the dialog is open.

    <a id="figure.auswahlfeld.datum"></a>

    ![Image: Dialog screen date](images/Calendar.png)

    **Figure 9. Dialog Screen Date**

<a id="eingabefeld"></a>**Input field (text field)**
:   Any text or numbers can be entered in the input fields using the keyboard. To enter something, you must click in the desired input field with the left mouse button (for example [Figure 10, “Input Field”](2-2-general-definitions-and-glossaries.md#figure.eingabefeld "Figure 10. Input Field")).

    <a id="figure.eingabefeld"></a>

    ![Image: Input field](images/eingabefeld.png)

    **Figure 10. Input Field**

<a id="fenster"></a>**Window**
:   Additional windows are displayed in some [frames](2-2-general-definitions-and-glossaries.md#rahmen). The header with the window title is the window [tab](2-2-general-definitions-and-glossaries.md#reiterkarten). Multiple windows can be open at the same time in one frame and can be selected through their respective tabs.

    Windows can also be moved and arranged next to each other ([Figure 11, “Arranging windows”](2-2-general-definitions-and-glossaries.md#figure.fenster_anordnen "Figure 11. Arranging windows")). To do this, move the mouse cursor over a window tab and then press and hold down the left mouse button. The window ([1a](2-2-general-definitions-and-glossaries.md#figure.fenster_anordnen "Figure 11. Arranging windows")) can now be moved when the mouse button is pressed down. If the window is moved to a position at which it can be saved, a black arrow will appear instead of the mouse cursor ([1b](2-2-general-definitions-and-glossaries.md#figure.fenster_anordnen "Figure 11. Arranging windows")) and the new position of the window will be shown as a dotted line ([1c](2-2-general-definitions-and-glossaries.md#figure.fenster_anordnen "Figure 11. Arranging windows")). Release the left mouse button to save the window at the displayed location.

    <a id="figure.fenster_anordnen"></a>

    ![Image: arranging windows](images/Fenster_anordnen.png)

    **Figure 11. Arranging windows**

    If the mouse cursor assumes the shape of the folder behind the window being moved, ![](images/Ordner.png)release the left mouse button to place the window in front of the other windows in a frame (such as [1a](2-2-general-definitions-and-glossaries.md#figure.fenster_anordnen "Figure 11. Arranging windows")).

    <a id="table.hinweis.fenster_und_editor"></a>

    If the layout of windows and frames were changed, the display can be reset to the **original layout** in ODIS Creator. To do this, click on the **View** [button](2-2-general-definitions-and-glossaries.md#schaltflaeche) ([3](2-2-general-definitions-and-glossaries.md#figure.fenster_anordnen "Figure 11. Arranging windows")) in the [menu bar](2-2-general-definitions-and-glossaries.md#menuleiste) and the **Reset view** button in the [selection menu](2-2-general-definitions-and-glossaries.md#auswahlmenu).

<a id="kontextmenu"></a>**Context menu**
:   Unlike a [selection menu](2-2-general-definitions-and-glossaries.md#auswahlmenu), a context menu is opened by clicking with the **right** mouse button (for example, [Figure 12, “Context menu”](2-2-general-definitions-and-glossaries.md#figure.kontextmenu "Figure 12. Context menu"))

    <a id="figure.kontextmenu"></a>

    ![Image: context menu](images/kontextmenu.png)

    **Figure 12. Context menu**

<a id="kontrollfeld"></a>**Checkbox**
:   Clicking on the checkboxes will activate or deactivate them (for example [Figure 13, “Checkbox”](2-2-general-definitions-and-glossaries.md#figure.kontrollfeld "Figure 13. Checkbox")).

    <a id="figure.kontrollfeld"></a>

    ![Image: checkbox](images/kontrollfeld.png)

    **Figure 13. Checkbox**

<a id="menuleiste"></a>**Menu Bar**
:   The menu bar is located in the upper window bar ([Figure 14, “Menu Bar”](2-2-general-definitions-and-glossaries.md#figure.menuleiste "Figure 14. Menu Bar")). The entries in the menu bar are [buttons](2-2-general-definitions-and-glossaries.md#schaltflaeche). Clicking on this type of button will open a corresponding [selection menu](2-2-general-definitions-and-glossaries.md#auswahlmenu). The amount of selectable buttons in the menu bar and the related selection menus is dependent on the current program view and can vary from that.

    <a id="figure.menuleiste"></a>

    ![Image: menu bar](images/menuleiste.png)

    **Figure 14. Menu Bar**

<a id="optionsfeld"></a>**Option field (radio button)**
:   An option field consists of multiple elements, of which only one can be active (for example, [Figure 15, “Option Field”](2-2-general-definitions-and-glossaries.md#figure.optionsfeld "Figure 15. Option Field")). The selection of an element will automatically deselect all other elements.

    <a id="figure.optionsfeld"></a>

    ![Image: option field](images/optionsfeld.png)

    **Figure 15. Option Field**

<a id="rahmen"></a>**Frames**
:   Frames are sections that are separated by a border. The frame is highlighted with the **order navigator** in the image ([Figure 16, “Frames”](2-2-general-definitions-and-glossaries.md#figure.rahmen "Figure 16. Frames")). The size of the frame can be changed or they can be hidden. To do this, move the mouse cursor over the edge of a frame, until a double arrow appears instead of the mouse cursor. Hold down the left mouse button and pull the frame in the desired direction until it reaches the desired size of the frame is hidden. To show a frame that is hidden, pull it back in the opposite direction.

    <a id="figure.rahmen"></a>

    ![Image: frame](images/rahmen.png)

    **Figure 16. Frames**

    If the layout and sizes of frames were changed, the display can be reset to the **original layout** in ODIS Creator. To do this, click on the **View** [button](2-2-general-definitions-and-glossaries.md#schaltflaeche) in the [menu bar](2-2-general-definitions-and-glossaries.md#menuleiste) and the **Reset view** button in the [selection menu](2-2-general-definitions-and-glossaries.md#auswahlmenu).

<a id="reiterkarten"></a>**Tabs**
:   The header with the title of a [window](2-2-general-definitions-and-glossaries.md#fenster) is given as the "tab". If multiple windows are opened in one frame, the tabs can be displayed behind each other. By clicking on a tab, the respective window will come to the front of the display ([1](2-2-general-definitions-and-glossaries.md#figure.reiterkarten "Figure 17. Editor with Tabs")). If more windows are open than can be displayed, a small double arrow **>>** will be displayed on the right side with the number of hidden tabs ([3](2-2-general-definitions-and-glossaries.md#figure.reiterkarten "Figure 17. Editor with Tabs")). Clicking on this will open a [selection menu](2-2-general-definitions-and-glossaries.md#auswahlmenu) with the window titles of the open tabs.

    <a id="figure.reiterkarten"></a>

    ![Image: editor with tabs](images/Editor.png)

    **Figure 17. Editor with Tabs**

    Double clicking on a window tab will expand it to the full size of the program. Double clicking on the tab again will restore the previous view.

<a id="schaltflaeche"></a>**Button**
:   Buttons are used by clicking on them, such as ![](images/abbrechen.png) or ![](images/plus.png) or the buttons in the [menu bar](2-2-general-definitions-and-glossaries.md#menuleiste) ![](images/menuleiste.png).

<a id="symbolleiste"></a>**Tool Bar**
:   The tool bar is the program toolbox. Tool bars are usually located under the [menu bar](2-2-general-definitions-and-glossaries.md#menuleiste) or in the header or footer lines of a [frame](2-2-general-definitions-and-glossaries.md#rahmen). The most important operational commands are displayed as symbols here ([Figure 18, “Tool Bar”](2-2-general-definitions-and-glossaries.md#figure.symbolleiste "Figure 18. Tool Bar")). Symbols are [buttons](2-2-general-definitions-and-glossaries.md#schaltflaeche). The amount of them depends on the current program view and therefore can vary. The commands can also be reached through the standard menu.

    <a id="figure.symbolleiste"></a>

    ![Image: tool bar](images/symbolleiste.png)

    **Figure 18. Tool Bar**
