6.3. Text request

In the Order managementview, click on the Text request action element. The “Text request” view shows the current text requests from the text author’s view.Only the “Author” basic role has access to the text request view.

Figure 142. Text Request Order Management


The individual text requests are shown in a tree navigation view. All information that was entered for the text request is displayed at the right next to the navigation tree. Everything except for the comments field and the field for requested text is read-only.

Figure 143. Text Request Editor


The author of a text request can perform the following actions:

  • Text request:Definitively accept. The author can definitively accept a text request after it has already been accepted by the Text Administrator.

  • Declining a text request:accept. The order will be discarded.

  • Text request:discard. The order will be discarded.

  • Declining a text request:objection. A reason for this must always be entered in the comments.

  • Text request:delete. This removes the text request from the process chain. To ensure that no orders that may still have open actions are deleted, the author can only delete new orders that have the status “accepted”, “declined”, “discarded”, “substitution error”, and “complete”, or change orders that have the status “declined”, “discarded”, and “complete”.

  • Reorder a text order with substitution error. This will reset the text order to the “Approved” status in order to trigger a new round of automatic replacement.
