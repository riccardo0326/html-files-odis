#### <a id="sgbaum.import"></a>7.7.1. Importing Control Module Descriptions

While control module descriptions are importing, the old, actual status of the control module descriptions is displayed. The new status is displayed only after the import is complete. Further information about the import of control module descriptions can be found under [Section 5.4, “Importing Control Module Descriptions”](5-4-importing-control-module-descriptions.md "5.4. Importing Control Module Descriptions").

##### <a id="sgbaum.import.gueltigkeit"></a>7.7.1.1. Validity of Control Module Descriptions

UDS control module descriptions are will be invalid for the same vehicle project due to the import of a new version of the control module description. When there is a later revision of the editorial object to be used, the author must select an alternative control module description, if an automatic replacement is not possible.

##### <a id="sgbaum.import.in.bearbeitung"></a>7.7.1.2. Early Import of Control Module Descriptions

It is possible to import generic function codes that work with vehicle projects and control module descriptions not yet released for the market. Control module descriptions imported with this method are then in the **„in processing“** status. The release of control module descriptions occurs through a regular import based on the applicable test criteria there. A manual release of control module descriptions by a user is explicitly **not** intended.
