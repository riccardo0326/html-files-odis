#### <a id="Funktionstesteditor.Debugger.Breakpoints_Erzeugen"></a>7.15.3. Creating Break Points

A break point represents an interruption in the process. If the function test editor encounters a break point in the process, the process interrupted and information about the current environment can be viewed and changed.

Break points can be created at nodes using the context menu with the entry "Break point on/off". See also [Figure 551, “Creating a Break Point”](7-15-3-creating-break-points.md#figure.Funktionstesteditor.Debugger.Breakpoints_Erzeugen "Figure 551. Creating a Break Point").

<a id="figure.Funktionstesteditor.Debugger.Breakpoints_Erzeugen"></a>

![Image: creating a break point](images/fte_breakpoint_erstellen.png)

**Figure 551. Creating a Break Point**

A red bug appears in that location that visually represents the break point. See also [Figure 552, “Break Point”](7-15-3-creating-break-points.md#figure.Funktionstesteditor.Debugger.Breakpoint_Kaefer "Figure 552. Break Point"). Clicking on the "Break point on/off" entry again deactivates the break point and the bug turns gray.

![Image: break point](images/fte_breakpoint_kaefer.png)

**Figure 552. Break Point**
