7. ODIS Creator Editing Process

The Editing module is the central component of the ODIS Creator editing system. It serves not only for the creation and management of editorial objects for the diagnosis, but rather also ensures their consistency, and provides the translation with translation candidates, has interfaces for a standard test language and the editing systems in context (for example, LIVAS 3, MOVE, etc.). It integrates internal and external content editors (such as the function test editor, Epic as XML editor, hotspot editor, etc.) and has interfaces for order management and administration as well as for supplying.

The editing module is assigned to the author role. The following views are available to the author:

IllustrationviewDescription
Equipment NetworkEquipment characteristics for the vehicle models are described for the individual brands in the equipment network.
Knowledge baseA knowledge base contains a diagnostic object tree, usually for a specific vehicle model or version.
Function LibraryThe function library refers to all functional objects.
Control Module TreeThe control module tree refers to all objects that describe the properties of control modules.
DTC Memory TreeThe DTC memory tree contains all available DTC memory entries in the group as DTC objects.
Vehicle projectThe vehicle project tree contains all available vehicle projects in the group. All vehicle projects are sorted by brand in this view.
Supplemental documentsA supplemental document is an editorial object that can be assigned to one or more diagnostic objects, a function test or other supplemental documents through an internal documentation reference.
XML TemplatesAn XML template is an editorial object that can be assigned to one or more test sequences of function codes through an action element, such as Read XML or Write XML/XML IA.
