6.2.5. Releasing Editorial Objects

In ODIS Creator, there is only one option for releasing editorial objects: the risk release. The author takes all responsibility for the accuracy of the editorial object.

6.2.5.1. Releasing Editorial Objects in Order Management

In the Order management view, click on the Orders action element (see Section 6.2, “Orders” ). In the order navigator, the order step with editorial objects that are released should be double clicked on. The editor will open to editing. A list of editorial objects, that were processed within the order step, will be displayed in the editor. Using the 'Risk release' filter button, the display can be reduced so that only editorial objects are displayed that fulfill the conditions for a risk release.

To select editorial objects that should be risk-released, click on them while pressing the Shift or Ctrl button in order to mark them. Click on the Edit and Risk release buttons in the menu bar or use the corresponding buttons from the context menu to access the marked editorial objects (Figure 125, “Releasing Editorial Objects in Order Management (Risk Release)”). All selected editorial objects then have the status "risk release".

Figure 125. Releasing Editorial Objects in Order Management (Risk Release)


6.2.5.2. Releasing Editorial Objects in Order Management

Select the editorial objects that should be risk released in the Editing view in a navigator. You are able to select multiple editorial objects that should be risk-released by pressing down the Shift or Ctrl button and clicking on the objects consecutively in order to mark them. Click on the Edit and Risk release buttons in the menu bar or use the corresponding buttons from the context menu to access the marked editorial objects (Figure 126, “Releasing Editorial Objects in Editing (Risk Release)”). All selected editorial objects then have the status "risk release".

If an object is not yet confirmed completed, the Complete and Release function can be used. This combines the completion and the risk release, by first performing the completion and then the risk release.
