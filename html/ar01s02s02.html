2.2. General Definitions and Glossaries

Fundamental and recurring terms and operations are explained in more detail in this section.

Lock button (toggle button)

Lock buttons are used by clicking on them, such as Figure 5, “Lock Button”. They will be activated or deactivated by clicking on them and will then maintain the selected status until the next input.

Figure 5. Lock Button


Selection field (combo box) and selection list

The program makes entries which the user can select available through so-called selection fields (Figure 6, “Selection Field”).

Figure 6. Selection Field


If a selection field (Figure 6, “Selection Field”) is clicked on, a selection list will open (Figure 7, “Selection List”). Through the arrow keys or can be navigated to the top or to the bottom. Alternatively, the scroll bar (marked in green) can be moved. Click on the desired entry to apply it.

Figure 7. Selection List


Selection Menu

A selection menu is a list of multiple buttons, that is opened by clicking a superordinate button (for example Figure 8, “Selection Menu”). The buttons in the selection menu are accessed by clicking on them. If you click in an area outside of the open selection menu, the selection menu will close.

Figure 8. Selection Menu


Dialog screen

An example for a dialog screen is the date input (Figure 9, “Dialog Screen Date”). Here the month and year can be changed by clicking on the month/year. However, the day is applied by clicking directly on it. By pressing the OK button, the selected value will be applied and the dialog will close. By pressing Cancel, the dialog will close without applying the value. In a standard scenario, the dialogs act as so-called modal dialogs. This means the application is blocked for entries in other elements as long as the dialog is open.

Figure 9. Dialog Screen Date


Input field (text field)

Any text or numbers can be entered in the input fields using the keyboard. To enter something, you must click in the desired input field with the left mouse button (for example Figure 10, “Input Field”).

Figure 10. Input Field


Window

Additional windows are displayed in some frames. The header with the window title is the window tab. Multiple windows can be open at the same time in one frame and can be selected through their respective tabs.

Windows can also be moved and arranged next to each other (Figure 11, “Arranging windows”). To do this, move the mouse cursor over a window tab and then press and hold down the left mouse button. The window (1a) can now be moved when the mouse button is pressed down. If the window is moved to a position at which it can be saved, a black arrow will appear instead of the mouse cursor (1b) and the new position of the window will be shown as a dotted line (1c). Release the left mouse button to save the window at the displayed location.

Figure 11. Arranging windows


If the mouse cursor assumes the shape of the folder behind the window being moved, release the left mouse button to place the window in front of the other windows in a frame (such as 1a).

Note: Open windows in the editor can now be moved inside of the right frame (2).

If the layout of windows and frames were changed, the display can be reset to the original layout in ODIS Creator. To do this, click on the View button (3) in the menu bar and the Reset view button in the selection menu.

Context menu

Unlike a selection menu, a context menu is opened by clicking with the right mouse button (for example, Figure 12, “Context menu”)

Figure 12. Context menu


Checkbox

Clicking on the checkboxes will activate or deactivate them (for example Figure 13, “Checkbox”).

Figure 13. Checkbox


Menu Bar

The menu bar is located in the upper window bar (Figure 14, “Menu Bar”). The entries in the menu bar are buttons. Clicking on this type of button will open a corresponding selection menu. The amount of selectable buttons in the menu bar and the related selection menus is dependent on the current program view and can vary from that.

Figure 14. Menu Bar


Option field (radio button)

An option field consists of multiple elements, of which only one can be active (for example, Figure 15, “Option Field”). The selection of an element will automatically deselect all other elements.

Figure 15. Option Field


Frames

Frames are sections that are separated by a border. The frame is highlighted with the order navigator in the image (Figure 16, “Frames”). The size of the frame can be changed or they can be hidden. To do this, move the mouse cursor over the edge of a frame, until a double arrow appears instead of the mouse cursor. Hold down the left mouse button and pull the frame in the desired direction until it reaches the desired size of the frame is hidden. To show a frame that is hidden, pull it back in the opposite direction.

Figure 16. Frames


If the layout and sizes of frames were changed, the display can be reset to the original layout in ODIS Creator. To do this, click on the View button in the menu bar and the Reset view button in the selection menu.

Tabs

The header with the title of a window is given as the "tab". If multiple windows are opened in one frame, the tabs can be displayed behind each other. By clicking on a tab, the respective window will come to the front of the display (1). If more windows are open than can be displayed, a small double arrow >> will be displayed on the right side with the number of hidden tabs (3). Clicking on this will open a selection menu with the window titles of the open tabs.

Figure 17. Editor with Tabs


Double clicking on a window tab will expand it to the full size of the program. Double clicking on the tab again will restore the previous view.

Button

Buttons are used by clicking on them, such as or or the buttons in the menu bar .

Tool Bar

The tool bar is the program toolbox. Tool bars are usually located under the menu bar or in the header or footer lines of a frame. The most important operational commands are displayed as symbols here (Figure 18, “Tool Bar”). Symbols are buttons. The amount of them depends on the current program view and therefore can vary. The commands can also be reached through the standard menu.

Figure 18. Tool Bar


Note: Each symbol is provided with an explanation of the function (tool tip). The tool tip appears when the mouse cursor is held over the corresponding symbol for a brief period without moving, and disappears automatically as soon as the mouse cursor moves again.
