8.10.3. Placeholder

The placeholders, that can be specified when creating a test, and a precise description of their values are given in the following table.

PlaceholderValue
BrandThe name of the preselected brand.
Test dateThe date on which the test object was created; in the format DD.MM.YYYY.
Test timeThe time that the test object was created, in 24-hour format hh.mm.ss.
Baseline versionFor tests that are started after a baseline is created, the version of the respective baseline is entered here.
Test type

  • full_product_baseline

  • incr_product_baseline

  • testbaseline

  • rolloutbaseline

  • full_enddate

  • incr_enddate

Test trigger

  • "manual" for manual tests

  • "automatic" for automatic tests

Test sort

The following tests simulate a baseline creation:

  • Complete production baseline

  • Incremental production baseline

  • Test Baseline

  • End date complete

  • End date incremental

The "simulated" value is entered in this case.

For tests that are started after a baseline is created, the "real" value is entered here.

Number of days after the deadlineFor tests started automatically after the deadline is reached, the word amount is replaced by the entered days after deadline. The word deadline is replaced by the given deadline in the format DD.MM.YYYY. Example: 3_days_after_enddate_01.07.2016
Baseline market nameFor tests that are started after a baseline is created, the market name of the respective baseline is entered here.
Baseline dateFor tests that are started after a baseline is created, the creation date of the respective baseline is entered here.
Baseline timeFor tests that are started after a baseline is created, the time at which the respective baseline was created is entered here in a 24-hour format (hh.mm.ss).
