5.3.1. Assignment of Technical Rights

5.3.1.1. Assignment of Views

The most basic functional right for a user role is which views they are allowed to use. This assignment is performed in the Rights management navigator (Figure 60, “Rights Management Navigator”) in a tree structure, which contains all of the available roles on the first level. The permitted views for the respective roles and an entry that is always available (General function rights) are displayed on the second level.

Figure 60. Rights Management Navigator


The Views... menu item can be accessed via the context menu for a role displayed in the rights management navigator (Figure 61, “Views Context Menu Item”). By activating or deactivating checkboxes, additional views can be assigned to roles, or views previously allowed can be revoked (Figure 62, “Available Views”).

Figure 61. Views Context Menu Item


Figure 62. Available Views


5.3.1.2. Allocation of Functional Rights (Function Editor)

In order to display the allocation of rights to the individual functions for a roll, the corresponding entry in the rights management navigator must be clicked on (Figure 60, “Rights Management Navigator”). The corresponding function editor will open for display in the right section (Figure 63, “Function Editor”). To change the allocation of rights, the editor must be opened in edit mode.

Another editor can be displayed using the rights management navigator or additional editors can be opened for editing. The role and view, that the function editor is opened in, is displayed at the top of the function editor. This display cannot be changed.

Figure 63. Function Editor


By activating or deactivating checkboxes, additional functional rights can be given to or removed from a role (Figure 63, “Function Editor”). The function names cannot be changed and only internal names are available in ODIS Creator. This is generally an abbreviation of the English name of the full function name.

In some tables there are References to object types in the right column, that are allocated to the respective function. These references are only active if the Editor being edited is open.
