5.2.6. Location Data

The location data entry in the master data navigator (2) is managed by the location administrator.

DataDescription
Teams

In this editor the teams are created for the specific locations and a corresponding role is assigned. The name of the team and the description can be freely selected for this. The system name is automatically generated and is stored as a unique ID permanently in the system after it is saved for the first time. When creating a new team, a roll is selected through a selection list from the group data. The roll cannot be changed for an existing team.

Furthermore, the administrator teams can activate and deactivate, as well as assign teams from a user list or delete relations. Multiple selections are possible in the selection menu by pressing the Ctrl or Shift key. Saved teams cannot be deleted by the administration, but rather can only be deactivated.

User

In this editor, the users are created for the individual locations. The Administrator can deactivate users. The Administrator can also delete deactivated users. The name of the deleted user will remain in the system and the user can be restored if it is created again using the same name (case sensitive).

Deleted users are no longer shown in ODIS Creator, for example in the object properties.

The administrator can also activate or deactivate the “Service” checkbox for each user. If the maintenance switch in vaudes_server.properties is activated, only users for whom the "Service” checkbox is activated can log into the OC.

Bridge languagesBridge languages aid in the cost-efficient translation into target languages. If a source document is in a language with high translation costs, the translation will first be done in a cost-effective language, such as English. In turn, this translation serves as the source language to be translated into the corresponding target language. Bridge languages are created separately for the individual locations using the selection lists. One bridge language can be selected per target language.
Clone languagesClone languages aid in the cost-efficient translation into target languages. If translations are identical for multiple target languages, then only one translation is needed. This target language is called a clone language. The rest of the languages are automatically populated with the content of the clone language. These target languages that are automatically populated are called cloned languages. Clone languages are managed separately for the individual locations.
SessionsThe Sessions editor is for the management of active user sessions. The administrator can issue the current active sessions per location, end active sessions, lock and unlock users, as well as send short text messages to an active user.
User lockThe administrator can unlock locked objects in this editor. As soon as an object is edited, the system locks this object and prevents any concurrent access by another user. System failures or other situations may cause this lock to not reset after completing the edit. When unlocked, the object can return to the editing state.
Organizational unitIn this editor, the group administrator can record and manage the names of organization units. These organizational units can be assigned to the users in the system.
