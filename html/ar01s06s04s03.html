6.4.3. Sorting Translations

Sorting translation packages or requirements can be done through the button (2) in the translation navigator (see Figure 150, “Translation Navigator with Buttons”).

The sorting dialog is pictured in Figure 152, “Sorting Dialog for Translation Objects”. A sorting of translation requirements or translation packages can be done separate from each other there. You must specify an attribute and an order (ascending/descending) for the sorting. The sequence is always given in "ascending order" at first.

Figure 152. Sorting Dialog for Translation Objects


Confirm the dialog with "OK", and the leaf nodes currently open in the tree will be sorted according to the set criterion. If other leaf nodes are displayed by opening sections of the tree while sorting is active, then these will also be automatically sorted accordingly. If the dialog is closed with the "Cancel" button, then the initial alphanumeric sorting is maintained.

The active sorting only has effects on the leaf nodes of the respective tree sections, therefore any displayed structure nodes (location, target language) have the initial alphanumeric sorting. Furthermore, the active sorting has no influence on the sorting in the editor tables.

The list of possible attributes permitted for the respective leaf nods is given in the following.

Translation requirements can be sorted according to the following attributes:

  • Location

  • Source language

  • Target language

  • Target date of the latest start of translation

Translation packages can be sorted according to the following attributes:

  • Location

  • Source language

  • Target language

  • Package creation date

  • Package target date

  • Package number

  • Package status
