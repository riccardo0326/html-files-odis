5.7.1. Remote Diagnosis Export Navigation Tree

The navigation tree for remote diagnosis exports shows all available Remote diagnosis exports in a tree structure. You have the option to display the properties of remote diagnosis exports (one mouse click).

The remote diagnosis exports are hierarchically structured in the tree based on their brand and the year they were created. Below the year nodes, there is another structural level for the “type” of preparation that should be used to generate the exports. In addition to displaying exports that have run, the navigator also provides the option of configuring automatic export jobs. The group administrator can configure all brands and sees exports for all brands. The location administrator can only configure brands for his specific location and only sees exports for those brands.

Note: the filtering of the brand nodes is based on the complete brand list for the respective user. He can also see exports for brands for which he is not the location administrator if he has another role.

The status options Created, In processing, Completed and Error are permitted for remote diagnosis exports.

All remote diagnosis exports can be deleted from the database individually or by selecting multiple exports using the Delete context menu. If possible, the export results will also be deleted in the save location.
