7.4.2. Creating Additional Basic Features

You can create additional basic featuresin the equipment network navigator, in which you select New subnode in the context menu, and then the vehicle model depending on the respective structure level, model year, variantor engine. This command can also be reached through the Insert selection menu. After that, the view New element: opens in the editor with the respective basic feature. In addition to the attributes already listed above, it can be specified for these basic features, if the CAN diagnosisaffects which Read-time for the constructionand the code delayshould apply. The UMB availabilitydefines the accessibility of the basic features for the UMB dealers.

For the vehicle model, version, and engine basic features, there is also the option to formulate a Carport inquiry. This is three characters long; four characters are also permitted for the engine. As characters, you can use the letters a-z, A-Z (the system is not case-sensitive) and the digits 0-9. The character "?" can be used as the wildcard symbol for one or more items. You can combine multiple carport inquiries to one basic feature using the "+" character. (see Figure 238, “Vehicle Recognition - Model Name”)

Using the Add markets function in the context menu of a brand or a basic feature, new market allocations can be transferred to this object all editorial objects directly or indirectly referenced by it. These market allocations will then automatically trigger any new translations needed, if languages are referenced by this market, in which the editorial object has not been available yet. You can find additional information about this under Section 6.1.2, “Tasks for Translation Coordinators” and Section 6.4, “Translation”. If you want to add only one brand or one basic feature exclusively for a new market, then you must apply this adjustment via a change in the market availability table in the object editor (see Figure 233, “Market Availability”).

Engines can also give an exact reference to a diagnostic object in the knowledge base, that references the engine of the superordinate vehicle model. Engines have additional attributes. This is Erase DTC memory (broadcast)to delete the DTC memory via broadcast, when exiting the OBD control modules. Wait time after switching on ignitionto define the wait time for the availability of control modules after switching the ignition on relating to the broadcast. Wait time after deletingto define the wait time for reading the DTC memory in the control modules, if the DTC memory was deleted via broadcast. OBD dieselto indicate if it is an EU5 vehicle with a diesel engine and therefore OBD-relevant measured values can be found at different locations than on a gasoline engine, in order to check the readiness code when exiting.
