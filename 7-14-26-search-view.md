#### <a id="Funktionstesteditor.Zusaetzliche_Views.Suche_View"></a>7.14.26. Search (View)

You can search for content of the function test using this view. Content can also be replaced. The search is always based on the function test that is currently active. Results are displayed in the context in which they were found. This includes the name of the command as well as a match highlighted in color within the surrounding context text (if available). The [Figure 544, “Search View”](7-14-26-search-view.md#figure.Funktionstesteditor.Zusaetzliche_Views.Suche_View.AnsichtSuche "Figure 544. Search View") shows the view with an example of a search.

<a id="figure.Funktionstesteditor.Zusaetzliche_Views.Suche_View.AnsichtSuche"></a>

![Image: function test editor - additional views - search view](images/fte_suche_ansicht.png)

**Figure 544. Search View**

Double-clicking on a result marks the associated command in the function test editor and focuses on it. The arrow buttons can be used to scroll quickly through results lists. The last search result remains within an ODIS Creator session. If the function test on which the last search was based is not open, the system will try to open it. This works if the function test is present in the local user folder.

If the function test on which the last search was based is no longer open in ODIS Creator, this will be indicated in the Search view. The text is highlighted in color.

<a id="figure.Funktionstesteditor.Zusaetzliche_Views.Suche_View.AnsichtSucheMitMeldungFkcNichtOffen"></a>

![Image: function test editor - additional views - search view with indication](images/fte_suche_ansich_fkc_nicht_offen.png)

**Figure 545. Search View with Indication that Function Test is Not Open**

For example, if the last search result is based on a function test that is open in read-only mode, a red indicator will appear if the function test is open in write mode (and the read-only function test is closed). This sis because write and read-only function tests are based on two different local files. Repeat the search in write mode to obtain reliable results.

If a search result is selected in the table while the corresponding function test is not open, a prompt will appear, asking if the corresponding function test should be opened.

Any number of matches can be found for each command. They are shown in brackets. The individual results are separated by three dots (...). The number specified by the results list is based on the number of matches across all commands.

The search view can be opened in various ways:

- From the ODIS Creator toolbar
- In the menu under the Edit item
- In the function test editor context menu (test steps view and command page)
- Using the keyboard shortcut CTRL+F. Note: the keyboard shortcut opens the editing search function. If a function test is open and active, it will be used for the search in the function test editor. If you are in an XML view, the XML search will be opened.

The search criteria can be entered in the search view and the search can be limited to the following criteria types:

<a id="Funktionstesteditor.Zusaetzliche_Views.Suche_View.Kriteriumtyp"></a>

- Text: searches the translatable portions of the function test (such as message text and translation portions of expressions).
- Variable/keyword: searches for variables and keywords in declarations/definitions and in all usage areas (such as allocations and expressions). Note: the $ symbol can be used to find keywords.
- Print: searches for contents from printouts.
- Parameter: searches for function test parameters.
- All: combines all four criteria types. Note: with combinations, commands may be duplicated in the results list (once per criteria type).

Details about the number of hits per search criterion type and examples can be taken from section [Section 7.14.26.2, “Search Results in Function Codes”](7-14-26-search-view.md#fte.suche.fte "7.14.26.2. Search Results in Function Codes").

###### <a id="Funktionstesteditor.Suchoptionen"></a>7.14.26.1.1. Search options

Searches can be made case sensitive. You can also use the repeat metacharacters \* and ?:

- Star/Asterisk (\*): stands for any number of characters.
- Question mark (?): stands for any character.

The metacharacters can be used in any combination. To search for metacharacters themselves, for example to search for a question mark in a message, the corresponded character must be masked. Example: \? finds the question mark.

###### <a id="Funktionstesteditor.Suchoptionen.Expertenmodus"></a>7.14.26.1.2. Expert Mode

An expert mode is integrated to handle complex search inquiries. If it is activated, a regular expression in accordance with the Java specification can be used (http://docs.oracle.com/javase/6/docs/api/java/util/regex/Pattern.html). The search expression itself controls if the search is case sensitive in expert mode. The user is responsible for the correct syntax. Syntax errors are marked in the results output with a red symbol ([Figure 547, “Search View with Errors in Search Expression”](7-14-26-search-view.md#figure.Funktionstesteditor.Zusaetzliche_Views.Suche_View.ExpertenmodusFehler "Figure 547. ###### <a id="Funktionstesteditor.Suchoptionen.Ergebnisexport"></a>7.14.26.1.3. Export of Results

With the export function, the search results can be exported and saved locally in Excel format. The results list as well as the information from the search category and the search criteria are included in the export.

###### <a id="Funktionstesteditor.Suche.Ersetzen"></a>7.14.26.1.4. Replacement Function

Using the replacement function, you can automatically replace function test content. The replacement function always works in the active function test. The replacement is performed throughout the entire function test, as soon as the "Replace all" button has been pressed.

The criterion types print, text, variable/keyword and all will be available (refer to [Section 7.14.26, “Search (View)”](7-14-26-search-view.md#Funktionstesteditor.Zusaetzliche_Views.Suche_View.Kriteriumtyp)). The replacement is limited to the criterion type. A characteristic during the replacement of formatted text content: generally the replacement is done in text components with the same formatting. However, if the comparison text overlaps in two different formatted text portions, then the replacement is then performed. The replacement text is then inserted in the first formatted section.

The replaced locations including the meta information can be exported as an Excel file using the "Export" function. The "last action" field indicates the last replacement action. The last replacement is exported.

In the filter tab, you can exclude individual instructions and test steps from the replacement.

The following image shows the replacement tab in the search view.

<a id="figure.Funktionstesteditor.Zusaetzliche_Views.Suche_View.Ersetzen"></a>

![Image: Search with Replace Tab View](images/fte_suche_ersetzen.png)

**Figure 548. Search with Replace Tab View**

A replacement procedure creates an entry for each instruction in the problems view, in which it was replaced. You can use these entries to check the replacement result. The entry displays a note and is processed during the consistency check in editing. It disappears as soon as you have checked and confirmed the associated instruction in the dialog.

Note: the replace function may produce invalid function test content. For this reason, the syntax check will be triggered after a replacement procedure. The results are displayed directly in the problems view. The Undo function can be used to restore to the previous status.
