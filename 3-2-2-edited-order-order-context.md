#### <a id="bearbeiteter.auftrag"></a>3.2.2. Edited Order (Order Context)

It is displayed directly under the tool bar which order you currently have in editing ([Figure 25, “Order Context Bar”](3-2-2-edited-order-order-context.md#figure.auftragskontext "Figure 25. Order Context Bar")).<a id="d4e697"></a>. Since deadlines are attached to the orders that could relate to specific content, it is important that you always know which order you are editing diagnostic data for.

<a id="figure.auftragskontext"></a>

![Image: order context bar](images/Auftragskontext.png)

**Figure 25. Order Context Bar**

If no order is displayed, a query is given by ODIS Creator as soon as you would like to create or change diagnostic information in ODIS Creator. You can set or change the edited order in each view by clicking on the icon ![](<images/Setzt den Kontext.png>) and selecting the suitable order step within an order context.

The dialog to select a procedure context shows all of the order headers and order steps currently visible for you. For a better overview, you can access a filter to filter the entries. The filter is opened through a button in the context dialog. For this, the recognized filter from the navigation tree in the "orders" view is opened (see [Figure 26, “Filter Button in the Context Dialog”](3-2-2-edited-order-order-context.md#figure.grundlagen.kontextdialog.auftragsfilter "Figure 26. Filter Button in the Context Dialog")). You can find more information about using the filter under [Section 3.2.3.1, “Defining the filter”](3-2-3-navigator.md#navigator.filter.definieren "3.2.3.1. Defining the filter").

<a id="figure.grundlagen.kontextdialog.auftragsfilter"></a>

![Image: filter button in the context dialog](images/grundlagen_kontextdialog_auftragsfilter.png)

**Figure 26. Filter Button in the Context Dialog**

If you cannot find an applicable order, you always have the option to define an order yourself and to set it as the order context.
